# 🚀 دليل رفع المشروع أونلاين (Deployment Guide)

هذا الدليل يشرح لك خطوة بخطوة كيف ترفع مشروعك (Backend + Frontend + Database) ليصبح متاحاً للجميع عبر الإنترنت.

## 📋 نظرة عامة
مكونات مشروعك هي:
1.  **قاعدة البيانات (Database):** PostgreSQL مع إضافة PostGIS (للخرائط).
2.  **الخادم (Backend):** Node.js + Express + Socket.io.
3.  **الواجهة (Frontend):** React + Vite.

سنستخدم منصة **Railway.app** (الأسهل والأفضل لمشروعك) أو **Render.com**.
*نصيحة:* **Railway** هو الأفضل لمشروعك لأنه يدعم Socket.io بشكل ممتاز وتجهيز PostGIS فيه سهل جداً.

---

## 🛠️ الخطوة 1: تحضير الكود (GitHub)
يجب أن يكون مشروعك مرفوعاً على GitHub. إذا لم ترفعه بعد، قم بذلك الآن.
(تأكد أن مجلد `node_modules` غير مرفوع - وهذا مضبوط تلقائياً في ملف `.gitignore`).

---

## ☁️ الخطوة 2: حجز السيرفر وقاعدة البيانات (على Railway)

1.  أنشئ حساباً على [Railway.app](https://railway.app/).
2.  اضغط **New Project** -> اختر **Deploy from GitHub repo**.
3.  اختر مشروعك من القائمة.
4.  الآن سيطلب منك إعداد "Services". سنحتاج لإضافة قاعدة بيانات أولاً.
    *   اضغط **Right Click** في اللوحة الفارغة -> **Database** -> **PostgreSQL**.
    *   انتظر قليلاً ليتم إنشاء قاعدة البيانات.
    *   اضغط على بطاقة PostgreSQL -> انتقل لتبويب **Data** -> انسخ رابط الاتصال (**Connection URL**). ستحتاجه لاحقاً.

---

## ⚙️ الخطوة 3: إعداد الـ Backend (الخادم)

1.  في Railway، ستجد بطاقة تمثل مشروعك المرفوع من GitHub. اذهب إلى **Settings**.
2.  غير الـ **Root Directory** إلى `/backend` (لأن ملفات السيرفر داخل هذا المجلد).
3.  اذهب إلى تبويب **Variables** وأضف المتغيرات التالية (مهم جداً):

    | اسم المتغير | القيمة (Value) |
    | :--- | :--- |
    | `PORT` | `5000` (أو أي قيمة، Railway سيحددها تلقائياً غالباً) |
    | `DATABASE_URL` | الصق رابط قاعدة البيانات الذي نسخته في الخطوة السابقة |
    | `JWT_SECRET` | اكتب رمزاً سرياً طويلاً ومعقداً (مثل: `my-super-secret-key-123`) |
    | `CLIENT_URL` | رابط الفرونت إند (سنعرفه في الخطوة القادمة، اتركه فارغاً مؤقتاً أو ضع `*`) |
    | `NODE_ENV` | `production` |

4.  **تفعيل PostGIS:**
    *   تحتاج لتفعيل إضافة الخرائط في قاعدة البيانات.
    *   في Railway، اضغط على قاعدة البيانات -> تبويب **Query**.
    *   اكتب هذا الأمر واضغط Run: `CREATE EXTENSION IF NOT EXISTS postgis;`

---

## 🌐 الخطوة 4: إعداد الـ Frontend (الواجهة)

الآن سنرفع الواجهة. الأفضل رفعها على **Vercel** أو **Netlify** (مجاني وسريع جداً لـ React/Vite)، أو يمكنك رفعها كخدمة ثانية على Railway بنفس الطريقة. لنفترض أنك ستستخدم **Vercel**:

1.  اذهب إلى [Vercel.com](https://vercel.com) وسجل الدخول بـ GitHub.
2.  اضغط **Add New Project** -> اختر نفس المستودع (Repo).
3.  في إعدادات المشروع (Project Settings):
    *   **Framework Preset:** اختر `Vite`.
    *   **Root Directory:** اضغط Edit واختر مجلد `client`.
4.  في قسم **Environment Variables**، أضف التالي:

    | اسم المتغير | القيمة (Value) | وصف |
    | :--- | :--- | :--- |
    | `VITE_API_URL` | رابط الباك إند من Railway | (مثلاً `https://my-backend.railway.app`) |
    | `VITE_WS_URL` | رابط الباك إند من Railway | (نفس الرابط أعلاه) |

5.  اضغط **Deploy**.

---

## 🔗 الخطوة 5: الربط النهائي

1.  بعد نجاح رفع الـ Frontend على Vercel، ستحصل على رابط دومين (مثلاً `https://myapp.vercel.app`).
2.  عُد إلى **Railway** (الباك إند) -> وتحديداً لمتغيرات البيئة (Variables).
3.  حدث قيمة المتغير `CLIENT_URL` لتصبح رابط الـ Frontend الجديد (`https://myapp.vercel.app`).
4.  سيعيد Railway تشغيل السيرفر تلقائياً.

---

## ✅ مبروك!
الآن مشروعك يعمل أونلاين:
*   الموقع يفتح من رابط Vercel.
*   يتصل بالسيرفر على Railway.
*   ويحفظ البيانات والخرائط في قاعدة بيانات Railway.
